And one more thing I tried

The last two days on the train were definitely not the best I've had lately. Thanks to no internet and the baby which kept crying through the day. I found my solace in the "reverse_me.py" problem.

  • reversed_2.py + light explanation

To the left(converted to 3.5 raw - Copy.py) is a slightly modified version of the original code reverse_me.py ( modifications -- converted syntax to python 3.5 , removed the if statement in the end to check for the correct answer.) The modified code just shows us the encrypted version of the characters we entered and it also shows the state of the characters through various stages (nevermind the odd names I gave to the stages). To the right(reverse.py) is quite possibly the "inverse" of the one to the left.

As you can see, I first entered "abcde123" to the left, which was then converted to a hash like thingy. I then copy-pasted the hash like thingy to my own reverse.py and let it do its job. And voila ! I got the same string, but as characters in an array.

Do notice that the intermediate stages of the message through the different stages of encryption and the decryption are the same , but in reverse order of course. Further justifying the fact that it (kind of) works. The "array_entering_xor" to the left is the same as "UN-XORed array" to the right in reversed.py. The "string form" is also the same for both.

How I reversed each function is a long story, I'll write all about it very soon, after reverse.py actually decodes the message without any errors halfway down the original message. "reverse.py" is still a work in progress, I'm still trying to figure out a way through the errors. Hopefully I can make it work within the next few days.
